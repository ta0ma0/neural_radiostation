# neural_radiostation
Radiostation based on local music collection with LLM and TTS DJ.

> **Важно:** Данный README не является пошаговым руководством по инсталляции (Step-by-step tutorial). Это скорее высокоуровневое пояснение к архитектуре и логике сборки проекта.

## Описание проекта
Автономная интернет-радиостанция на базе локальной музыкальной коллекции с нейро-диджеем. Проект использует локальные LLM для генерации подводок к трекам и модель F5-TTS для синтеза естественного голоса ведущего. 

## Логика работы и компоненты

### 1. Подготовка данных
* **Индексация:** Перед запуском станции необходимо просканировать локальную коллекцию музыки. Скрипт `tools/collection_parser.py` создаст базу данных `music_collection.db` (путь до вашей директории с музыкой нужно указать в коде вручную).
* **Джинглы:** Отбивки и аудио-вставки лежат в `data/jingles`. В данном проекте они были сгенерированы с помощью ИИ-сервиса Suno (хватило Free tier).
* **Фоновый звук:** Используются атмосферные записи гула ЭВМ из музея Яндекса («Мелодии и ритмы ЭВМ») для создания нужного вайба.

### 2. Нейросетевой пайплайн
* **LLM-движок:** Текущая версия адаптирована под локальную LLM через LM Studio (`tools/ai_connector.py`). Так как API OpenAI-совместимо, можно легко переключиться на любой другой облачный или локальный сервис (Ollama, llama.cpp и т.д.).
* **Промпты:** Логика поведения диджея описана в `tools/journal_prompt_generic.py`. Формат позволяет легко перевести вещание на английский или любой другой язык, если ваша LLM это поддерживает.
* **Голос (TTS):** За генерацию и клонирование голоса отвечает F5-TTS (использовался форк `F5-TTS_RUSSIA`). В репозитории есть CLI-скрипт для генерации аудио с клонированием голоса: `tts_run.sh`. Подробнее смотрите в документации оригинального проекта.

### 3. Вещание и Фронтенд
* **Запуск:** Точка входа и запуск всего пайплайна — скрипт `play_music.py`.
* **Стриминг:** Аудиопоток выводится в сеть через `Icecast`.
* **Веб-интерфейс:** В качестве фасада выступает веб-терминал. Я использую форк [django-aws-terminal-websocket](https://github.com/agusmakmun/django-aws-terminal-websocket.git), но подойдут и аналоги. Ссылка на Icecast-поток встраивается во фронтенд `xterm.js`.
* **Логирование:** Все рабочие процессы станции пишут логи в файл `django-aws-terminal-websocket/dj_alyx_radio.log`, откуда они транслируются прямиком в веб-терминал зрителя.
